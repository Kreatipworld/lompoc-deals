## Data Format for DB Import

Each post maps to the following `blog_posts` table columns (recommend CTO implement):

```sql
CREATE TABLE blog_posts (
  id SERIAL PRIMARY KEY,
  slug VARCHAR(200) NOT NULL UNIQUE,
  title VARCHAR(500) NOT NULL,
  excerpt TEXT,
  body TEXT NOT NULL,
  category VARCHAR(100),
  meta_description VARCHAR(300),
  target_keywords TEXT[],
  internal_link_slugs TEXT[],
  category_page_links TEXT[],
  published BOOLEAN DEFAULT false,
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

Files `posts-1-25.json` and `posts-26-50.json` are ready for import once the CTO has the schema in place.
