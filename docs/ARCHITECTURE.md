# Lompoc Deals — Architecture
*Last updated: 2026-04-07*

---

## System Overview

```
Browser
  │
  ├─▶ Vercel CDN / Edge Network
  │     │
  │     ├─▶ Next.js App (App Router, Node.js runtime)
  │     │     ├─▶ Server Components (RSC)
  │     │     ├─▶ Server Actions (mutations)
  │     │     └─▶ Route Handlers (API)
  │     │
  │     └─▶ Static Assets (Vercel Blob CDN for images)
  │
  └─▶ External Services
        ├─▶ Neon Postgres (serverless HTTP driver)
        ├─▶ Stripe (billing)
        ├─▶ Resend (email)
        └─▶ Apify (Zillow scraper)
```

├── (auth)/             ← Unauthenticated only

│
├── dashboard/          ← Role: business (middleware gated)

## Data Layer

- **ORM**: Drizzle with `@neondatabase/serverless` HTTP driver
- **Schema**: `db/schema.ts` — single source of truth
- **Migrations**: `db/migrations/` — SQL files generated by `drizzle-kit`
- **Reads**: `lib/queries.ts` — centralized query functions (Drizzle select)
- **Writes**: `lib/biz-actions.ts`, `lib/auth-actions.ts`, `lib/admin-actions.ts`, etc. — Next.js Server Actions (form mutations)
- **Tracking**: `lib/tracking.ts` — view/click counter bumps

### Key query patterns
```typescript
// Approved active deals
where(and(eq(businesses.status, 'approved'), gt(deals.expiresAt, new Date())))

// Tier enforcement
if (existingDeals.length >= tierLimit) throw new Error('Upgrade required')
```

## Authentication

- **Provider**: Auth.js v5 credentials (email + bcrypt)
- **Session**: JWT stored in httpOnly cookie
- **Roles**: `local` | `business` | `admin` (stored in JWT)
- **Gates**: `middleware.ts` checks role + session for `/dashboard/*` and `/admin/*`
- **Session access**: `lib/viewer.ts` — unified `getViewer()` helper for server components

## Deployment

- **Platform**: Vercel (auto-deploy from main branch)
- **DB**: Neon serverless (HTTP, no connection pooling needed)
- **Cron**: Vercel scheduler (`vercel.json`)
  - `0 9 * * 6` — weekly digest (Saturday 9am UTC)
  - `0 6 * * *` — daily Zillow sync (6am UTC)
- **Images**: Vercel Blob (CDN-backed storage)
- **DNS**: GoDaddy domain → Vercel

## Future Architecture Notes

- **QR redemption**: Will need a short-lived token system (signed URLs or UUID codes stored in DB)
- **Featured listings**: Stripe add-on prices or one-time payments; needs a `featured_deals` table or a `featuredUntil` column on `deals`
- **Rate limiting**: Vercel Edge Middleware + upstash/ratelimit (KV-backed)
- **Analytics**: PostHog or GA4 event tracking; will require client-side script
- **Multi-city**: Add `city` field to `businesses`; scope all queries by city; use subdomain or path routing
